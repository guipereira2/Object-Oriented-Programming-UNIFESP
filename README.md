# Object Oriented Programming UNIFESP

Exercícios de Programação Orientada a Objetos da disciplina de POO da UNIFESP, implementados em C#.

## Listas

### Lista 1: Fundamentos de C#

| Exercício | Descrição |
|-----------|-----------|
| 1 | Hello World e entrada de dados |
| 2 | Operações aritméticas entre dois inteiros |
| 3 | Soma de números lidos em uma linha separados por espaço |
| 4 | Menu: cálculo de IMC e conversão Celsius → Fahrenheit |
| 5 | Criação de N arquivos `.txt` numerados em `dados/` |
| 6 | Leitura dos arquivos de `dados/` e soma dos valores |
| 7 | Cópia de arquivos de `dados/` para `backup/` |
| 8 | Lixeira: move arquivos com valor ≤ 1 e deleta os demais |
| 9 | Vetor `vec[i] = i²`: imprime todos, pares e soma dos ímpares |
| 10 | Busca de nome em array de strings com contagem e índices |
| 11 | Busca linear em `int[]`, retorna -1 se não encontrar |
| 12 | Leitura de duas matrizes, soma e salvamento em `.txt` |
| 13 | Normalização min-max por coluna de matriz e exportação para `.csv` |

### Lista 2: Programação Orientada a Objetos

| Exercício | Descrição |
|-----------|-----------|
| 1 | Classes `Pessoa` e `Aluno` com herança, encapsulamento e polimorfismo em `exibir_dados()` |
| 2 | Classes `Numero_Natural`, `Numero_Inteiro` e `Numero_Racional` com herança e validação |
| 3 | Extensão de `Numero_Racional` com `show`, sobrecarga de `==` e `+` |
| 4 | Classe `Carta` com encapsulamento e sobrecarga de `==`, `>` e `<` por peso |
| 5 | Classe `Baralho` com construção, embaralhamento e retirada aleatória de cartas |
| 6 | Instâncias de baralho italiano, inglês e inglês duplo usando `Baralho` |
| 7 | Classe `Baralho_de_Truco` com vira e atualização de pesos das manilhas |

### Lista 3: Análise de Código

| Exercício | Descrição |
|-----------|-----------|
| 1 | Identificação de erros em classe `A` com construtores duplicados e visibilidade |
| 2 | Análise de herança da classe `B` a partir de `A` corrigida |
| 3 | Classe `Q` (número racional) com getters, setters e validação do denominador |
| 4 | Uso de vetor de objetos `Q[]` e identificação de erros de inicialização |
